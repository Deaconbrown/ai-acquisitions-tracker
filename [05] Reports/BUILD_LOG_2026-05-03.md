## What Was Built
A fully automated AI acquisition news scraper that:
- Monitors 6 RSS news feeds every 2 hours
- Filters for AI company acquisition stories using keywords and company names
- Detects and skips duplicate articles by URL and headline similarity
- Saves matched stories to a CSV file on Google Drive
- Runs automatically in the cloud via GitHub Actions — no PC required
- Logs every run with full match, skip and error detail

### Stage 4 — Scraper Script (scraper.py)
Built iteratively across multiple runs:

Version 1 — Basic scraper
  - Reads sources.json for keywords, companies, feeds
  - Fetches each RSS feed using requests
  - Parses with BeautifulSoup
  - Checks headline and summary for keyword + company match
  - Saves matches to CSV with date, headline, summary, URL, feed source
  - Logs every action to scraper_log.txt

Version 2 — Duplicate detection
  - URL exact match check
  - Headline similarity check — 75% word overlap threshold
  - Common words excluded from comparison
  - SKIPPED log entries added for audit trail
  - Skipped count added to run summary

Version 3 — Reliability improvements
  - requests.Session with HTTPAdapter retry strategy
  - 3 retries, backoff factor 2, status codes 429/500/502/503/504
  - Timeout increased from 10s to 15s
  - Unicode fix: sys.stdout.reconfigure(encoding="utf-8")
  - warnings.filterwarnings("ignore") added

Version 4 — Google Drive integration
  - google-auth and google-api-python-client libraries used
  - personal_drive_token.pickle authenticated against arranwilliams@gmail.com
  - drive.file scope — creates and updates files owned by the app
  - get_drive_service() — loads and refreshes token automatically
  - upload_to_drive() — creates CSV on first run, updates on subsequent runs
  - Drive sync runs after every scraper run
  - acquisitions.csv confirmed live on Google Drive
  - File ID: 110_g-AuLvfKskFnyR96dWfB_OUW8Uodn

Version 5 — Cloud/local path detection
  - RUNNING_IN_CLOUD flag reads GITHUB_ACTIONS environment variable
  - Cloud paths use /tmp/ — local paths use Windows H:\ paths
  - DRIVE_TOKEN_PATH reads from environment variable with Windows fallback
  - download_csv_from_drive() added — downloads existing CSV before each run
  - Prevents duplicate entries across cloud runs where /tmp/ is wiped each time

## Issues Encountered and Fixes

| Issue | Root Cause | Fix |
|---|---|---|
| Reuters RSS feed dead | DNS resolution failure | Replaced with BBC Tech feed |
| playwright pip install failed | Environment config | Used python -m pip install instead |
| Node.js playwright installed first | Wrong version | Removed, installed Python version |
| Unicode box-drawing characters in log | Windows encoding | sys.stdout.reconfigure(encoding="utf-8") |
| Near-duplicate stories from different sources | Same story, different URLs | 75% headline similarity check added |
| Connection timeouts on 4 of 6 feeds | WAN instability | Retry logic with backoff added |
| BBC feed accidentally duplicated in sources.json | Manual edit error | Caught by terminal Claude before run |
| git global identity not set | Fresh git install | Configured Deaconbrown / arranwilliams25@hotmail.com |
| gitignore square brackets treated as regex | Git character class syntax | Escaped: \[02\] Data/ etc |
| gcloud CLI not installed | Google Cloud SDK not on PC | Used existing OAuth token instead |
| Drive token expired and wrong account | Token was for gaming Drive | Re-authenticated against arranwilliams@gmail.com |
| drive.file scope limitation | Cannot write to arbitrary folders | Acceptable — app creates and owns the file |
| Cloud runs wipe /tmp/ between jobs | GitHub Actions ephemeral storage | download_csv_from_drive() added as pre-run step |
| Push failed — workflow scope missing | gh CLI needed extra permission | gh auth refresh -h github.com -s workflow |
| Intermittent push failures throughout | WAN cable link-flapping | Retried manually — all succeeded eventually |
